# Design: Fix Number Formatting in PDF Reports

This document outlines the design to change the thousands separator of currency and amount values in generated PDF reports to use spaces instead of commas (e.g., `74 593.31` instead of `74,593.31`).

## Problem Description

1. In the Valterra RPM custom front page (`valterra_rpm/1-front-page.html`), numbers were printed as raw strings (e.g., `74593.31`) due to missing formatting filters.
2. Other PDF reports (Standard layout and Valterra RPM Summary/Detailed reports) used Django's built-in `intcomma` filter, which formats numbers with a comma thousands separator under the `en-us` locale.
3. The desired formatting is a space thousands separator and a dot decimal separator (e.g., `74 593.31`).

## Proposed Solution

1. **Custom `space_intcomma` Template Filter**:
   Implement a custom Django template filter `space_intcomma` in `app/core/templatetags/template_extras.py` that formats numeric values or string numbers to use space thousands separators while preserving the decimal portion.
   
2. **Template Integration**:
   Update all PDF-related HTML templates to use the new `space_intcomma` filter. This ensures all payment certificates generated by the system have uniform, correct formatting.
