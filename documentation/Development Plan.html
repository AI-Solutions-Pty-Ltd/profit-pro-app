Stage 1 – Initial development

Goal: Establish the technical and operational foundation for governance and claims/payment workflows.

Activities

  • Requirements gathering: Map process steps, decision points, and critical paths; identify all user roles (Client PM, Contractor, Inspector, Consultant, PA/PM, Client Rep).
  • System architecture: Define modules (Project Setup, Contractor Management, Inspection Scheduling, Claims & Payments), data flow, and integration points (e.g., compliance doc storage, inspector assignment).
  • Environment setup: Prepare development, test, and staging environments with CI/CD scaffolding.
  • Core data models: Create entities for Project, House, Contractor, Inspection, Claim, Payment, and relationships.
  • Security & access control: Implement role-based permissions, authentication, and audit logging baseline.
  • Baseline UI/UX: Wireframes and design tokens for project setup, inspection scheduling, and claim submission.

Deliverables

  • Approved requirements: Functional and non-functional scope with acceptance criteria.
  • Architecture pack: System and data flow diagrams; integration contract outlines.
  • Initial schema: Database schema with migrations.
  • Clickable wireframes: Key journeys across governance and claims/payment.
Foundation Security Architecture

Stage 2 – MVP

Goal: Deliver an end‑to‑end working version of governance and claims/payment critical paths.

Scope

  • Project governance core: Project and house setup, contractor and inspector assignment, inspection scheduling and results, payment trigger on pass.
  • Claims & payments: Contractor claim submission on approved template, consultant assessment, compliance docs, PM/PA/Client approvals, payment or amendment routing.
  • Reporting: Governance reporting in accordance to EPWP requirements
  • Notifications & tracking: Alerts for approvals, rejections, inspection outcomes, and payment status.
  • Audit trail: Immutable logs for reviews and compliance.

Testing

  • Quality gates: Unit, integration, and UAT with representative sample projects and claims.
  • Decision loops: Simulate rejection/rectification and resubmission flows for inspections and claims.

Deliverables

  • Deployed MVP: Production deployment with monitoring.
  • User onboarding: Role‑specific quick start guides and training sessions.
  • MVP documentation: Admin and user guides, runbooks.
E2E Flow UAT Go‑Live

Stage 3 – Post go‑live support & updates

Goal: Stabilize, optimize, and enhance the system based on real‑world usage.

Support & maintenance

  • Monitoring: Track performance, errors, and usage patterns; establish SLOs and incident response workflow.
  • Bug fixes: Triage and resolve issues via a prioritised backlog.
  • User feedback loop: Capture insights from contractors, inspectors, consultants, and client reps; feed into roadmap.
  • Enhancements: Reporting dashboards, mobile optimizations, external integrations as needed.
  • Scaling & security: Performance tuning, capacity planning, and security hardening.
  • Training & handover: Updated documentation and enablement for internal teams.

Deliverables

  • Operational reports: Monthly performance and feedback summaries.
  • Roadmap updates: Prioritized enhancements with release cadence.
  • Documentation set: Up‑to‑date technical and user materials.
